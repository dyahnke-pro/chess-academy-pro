### Real concerns (≠ sandbox / not pre-existing flake)

1. **`coach-endgame.spec.ts:81`** — `expect(recorder.pageErrors).toEqual([])` fails on hub mount. Page snapshot shows ALL 8 tabs present and the hub fully loaded. Some browser-side JS exception fires during boot. Endgame surface code wasn't touched today, so this is pre-existing. The cascade-skip of 25 subsequent tests means the endgame surface is **unverified end-to-end** in this run.
   - **Action**: needs a dedicated investigation pass to identify the exception and either suppress (if benign) or fix.

2. **`audit-reports/structural.md` flags 2 illegal arrows in `pro-dubov-italian.json`** (commit `46ad28b1`, today). The arrows `d5→e4` and `d5→a8` are concept-attack arrows showing the bishop's post-move pressure. Renderer draws them correctly (uses post-move position); audit script validates against pre-move FEN and so flags them as "not a legal move from d5 in the pre-move position." **False positive in the audit script**, not a data bug.
   - **Action**: optional — update `audit-structural.mjs` to check arrows against `fenAfter` for the moved piece's own concept arrows.

3. **`pro-annacramling-cow.json` + `pro-dubov-italian.json` have 7 empty-string annotations** flagged "critical" by `audit-data-quality.mjs`. Per CLAUDE.md narration rule #4 ("Silence is acceptable"), these are **intentional** — the moves are routine setup where voice should stay silent. **False positive in the audit script.**
